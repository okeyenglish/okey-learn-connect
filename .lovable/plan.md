
# Fix: Массовая привязка — все номера с ошибкой "не найдено"

## Проблема
Поиск клиентов по телефону использует точное совпадение (`phone.eq.79161234567`), но в базе данных телефоны хранятся в разных форматах (`+7 (916) 123-45-67`, `+79161234567`, `89161234567` и т.д.). Из-за этого ни один клиент не находится.

## Решение
Заменить точное сравнение (`eq`) на поиск по последним 10 цифрам через `ilike` — тот же паттерн, который используется во всём остальном проекте (webhooks, CRM, поиск клиентов).

## Технические детали

### Файл: `src/components/admin/BulkBranchReassign.tsx`

**Строки 104-108** — заменить URL поиска:

Было:
```
or=(phone.eq.${searchPhone},phone.eq.%2B${searchPhone},phone.eq.8${searchPhone.slice(1)})
```

Станет:
```
phone=ilike.%25${last10}%25
```

Где `last10` = последние 10 цифр нормализованного номера (без кода страны). Этот паттерн (`ilike.%${phone.slice(-10)}%`) используется повсеместно в проекте: в `telegram-webhook`, `max-webhook`, `LinkChatToClientModal` и других модулях.

Также убрать бессмысленную строку 105 (`const searchPhone = phone.startsWith('7') ? phone : phone` — всегда возвращает `phone`).
