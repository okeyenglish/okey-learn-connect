
# План: Убрать дублирующуюся шапку O'KEY ENGLISH со страниц

## Проблема

На странице `/install` (и, возможно, других) показывается **две шапки**:
1. **Основная навигационная шапка** O'KEY ENGLISH с меню (Главная, Программы, Филиалы и т.д.) — из компонента `Header.tsx` в `App.tsx`
2. **Внутренняя шапка страницы** Install с текстом "O'KEY ENGLISH" и кнопкой "Войти в CRM"

Это происходит потому что страница `/install` рендерится внутри основного layout'а (строки 300-639 в `App.tsx`), который добавляет `<Header />` ко всем страницам.

## Решение

Добавить условие для `/install` (и других standalone страниц) чтобы они рендерились без общей шапки и футера, т.к. у них есть свой полноценный layout.

## Технические изменения

### Файл: `src/App.tsx`

**1. Добавить проверку для страницы Install:**

```typescript
// После строки ~162
const isInstallPage = location.pathname === '/install';
```

**2. Добавить ранний return для Install страницы (после isOnboardingPage, ~строка 206):**

```typescript
// Install page has its own complete layout
if (isInstallPage) {
  return (
    <Suspense fallback={<LoadingComponent />}>
      <Routes>
        <Route path="/install" element={<Install />} />
      </Routes>
    </Suspense>
  );
}
```

**3. Удалить дублирующийся роут `/install` из основного блока Routes** (строки 577-581), чтобы избежать конфликтов.

## Дополнительные страницы для проверки

Если пользователь сообщает что проблема на других страницах тоже — нужно проверить:
- `/payment-success`
- `/payment-fail`
- `/employee/onboarding/:token`
- `/teacher/onboarding/:token`

Эти страницы тоже могут иметь свой layout и требовать аналогичного исправления.

## Ожидаемый результат

После изменений:
- Страница `/install` будет показывать только свою внутреннюю шапку с "O'KEY ENGLISH" и кнопкой "Войти в CRM"
- Основная навигационная шапка сайта НЕ будет показываться на этой странице
