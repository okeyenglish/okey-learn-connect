## ✅ ВЫПОЛНЕНО: Авто-рефреш JWT токенов для WPP интеграции

### Реализованные изменения

1. ✅ **`wpp-send/index.ts`** — добавлено чтение/сохранение JWT из БД
   - Читает `wppJwtToken` и `wppJwtExpiresAt` из settings
   - Проверяет валидность с буфером 60 сек
   - После отправки сохраняет обновлённый токен обратно

2. ✅ **`_shared/wpp.ts`** — добавлена специальная обработка `INVALID_API_KEY`
   - При ошибке 401 или "Invalid API key" выбрасывает понятную ошибку
   - Подсказывает использовать `force_recreate`

3. ✅ **`wpp-status/index.ts`** — уже имел JWT кеширование (проверено)

4. ℹ️ **`wpp-webhook/index.ts`** — не требует JWT (только принимает события)

### Следующие шаги

1. Задеплоить на self-hosted сервер: `cd /root && docker compose up -d --build`
2. Протестировать отправку сообщений через UI
