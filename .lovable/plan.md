
# Исправление: чаты без сообщений перекрывают актуальные

## Проблема
Клиенты, подгруженные через массовую привязку, получают дату `01.01` (из `last_message_at` или `created_at`) и поднимаются выше реальных активных чатов. У них нет сообщений ("Нет сообщений"), но они занимают весь экран.

## Решение
Добавить в сортировку чатов (строки 1302-1327 в `CRM.tsx`) правило: чаты без сообщений (`lastMessage === 'Нет сообщений'`) всегда опускаются ниже чатов с реальными сообщениями, даже если у них есть timestamp.

## Технические детали

### Файл: `src/pages/CRM.tsx`

В блоке `.sort((a, b) => ...)` (строки 1302-1327) добавить после проверки непрочитанных (строка 1323) и перед сортировкой по времени (строка 1326):

```typescript
// Чаты без сообщений — в конец списка
const aNoMessages = a.lastMessage === 'Нет сообщений';
const bNoMessages = b.lastMessage === 'Нет сообщений';
if (aNoMessages && !bNoMessages) return 1;
if (!aNoMessages && bNoMessages) return -1;
```

Порядок сортировки станет:
1. Закрепленные
2. Результаты поиска (люди vs сообщения)
3. Непрочитанные
4. **Чаты с сообщениями выше чатов без сообщений**
5. По времени последнего сообщения
